Les requetes HTTP sont toutes faites a l'aide de l'outil XMLHttpRequest

Listes des requetes :

- App.jsx
	- Element React Website, bloc conditionnel, ligne 26 > Premiere grosse requete, initialisation de la session
		- Je recupere de toute façon les profiles
		Si une session precedente a utilise le cookie pour garder la connection active, j'utilise les donnees du profil predemment connecté pour definir quel ladder et quelle liste de tournois je recupere
		Sinon, pong dans les 2 cas

- customWindows.jsx
	- Element React Settings, fonction validateChanges(), ligne 326 > Appel a la validation de changements eventuels dans la config du profil connecté
		- Je recupere le ladder et la liste des tournois du jeu auquel le joueur choisit de jouer, dans tous les cas, meme s'il a changé autre chose sans toucher a ça

	- Element React Login, fonction tryConnect(), ligne 546 > A la connection
		- Envoie une adresse e-mail et un mot de passe pour tenter de se connecter
		Renvoie un id
		Ensuite, je recupere la liste des profils pour etablir la friendList du profil qui vient de se connecter
		Je recupere aussi le ladder et les tournois du jeu configuré sur le profil

	- Element React Subscribe, fonction addUserToDb(), ligne 706 > A l'inscription
		- Envoi d'une adresse e-mail, d'un username et d'un mot de passe pour la creation d'un nouveau profil
		Renvoie un profil 'vierge' selon le modele du fil Discord

- other.jsx
	- Fonction loadProfile(), ligne 5 > A chaque affichage d'un profil, n'importe lequel, sauf a la connection
		- Je recupere la liste des profils et j'en extrais le profil que je veux afficher et ceux de ses amis

	- Element React Local, fonction getProfile(), ligne 167 > A la connection a une partie locale
		- Envoie une adresse e-mail et un mot de passer pour tenter de se connecter
		Renvoie {avatar, name, id} du profil correspondant



A terme, j'espere pouvoir lier tous ces elements a la DB avec useEffect() pour qu'il soient mis a jour en temps reel
