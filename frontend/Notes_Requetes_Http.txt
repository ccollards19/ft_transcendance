Les requetes HTTP sont toutes faites a l'aide de l'outil XMLHttpRequest

Listes edes requetes :

- customWindows.jsx
	- Element React Login, fonction login(), ligne 561 > A la connection, en fonction du nombre d'essais
    	- La fonction tryConnect() appelle la DB et lui fournit une adresse e-mail et un mot de passe pour la connection
    	Elle renvoie un int > 0 si la connection reussit, -1 si elle echoue
    	Juste apres, cet int est utilisé pour recuperer les donnees du profil correspondant (appel a la db)
	
	- Element React Subscribe, fonction subrscribe(), ligne 674 > A l'inscription, tant que l'utilisateur utilise une adresse deja existante
		- La fonction addUserToDb() envoie une adresse e-mail et un mot de passe a la DB, qui verifie que l'adresse n'est pas deja utilisee
		Elle renvoie un nouveau profil, d'apres le modele du fil Discord, avec le nouvel ID (int) comme valeur 'id'

	- Element React Profile, bloc conditionnel, ligne 156 > A chaue affichage du profil (sauf si profil === dernier profil visité)
		- Optimisation en back-end : Ne renvoyer que les profils pertinents (le profil consulté et ses amis)

	- Element React Settings, fonction validateChanges(), ligne 356 > A chaque changement de config
		- La fonction envoie une request 'POST' a la db pour modifier le profil en fonction de ses nouvelles configurations
		En parametre, la nouvelle config (qui aura forcement au moins un changement)

	- Element React Leaderboard, bloc conditionnel, ligne 497
		- (Pour le moment, a modifier plus tard) > Un appel au chargement du site, puis chaque fois que le state 'game' est modifié (dans Settings ou dans Play/Local)
		Renvoie les id des [up to] 50 meilleurs joueurs du site, pour le jeu choisi
		
	- Element React Play, bloc conditionnel, ligne 458 > A la connection
		- 2 requetes pour le prix d'une
		Appelle la DB pour obtenir la liste des challengers / challenged / tournaments, en fonction du contenu des arrays [game].challengers / challenged / tournaments du profil connecté

- other.jsx
	- Element React Ladder, bloc conditionnel, ligne 37 > Apres chaque appel de l'element Leaderboard
		- Appelle la db pour obenir les profils correspondants aux ID obtenus dans l'element Leaderboard

	- Element React Local, fonction getProfile(), ligne 154 > A chaque tentative de connection pour une partie en Local
		- Envoie une adresse e-mail et un mot de passe a la DB pour recuperer un profil pour une partie locale
		Renvoie un profil (name, avatar et spectate suffisent, ajouter device plus tard)
